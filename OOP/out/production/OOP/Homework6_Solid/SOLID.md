SOLID

1) Single Responsibility Principle (Принцип единственной ответственности).
- Один класс - одна функция. Функция целиком в одном классе.
- Класс Controller c 9 функциями разбил на разные классы (пакет Controller)
3) Open Closed Principle (Принцип открытости/закрытости).
- Если нужно нарастить функционал класса, то не нужно его менять, нужно его расширить и переопределить функции
4) Liskov’s Substitution Principle (Принцип подстановки Барбары Лисков).
- Объекты в программе можно заменить их наследниками без изменения свойств программы. Наследники должны возвращать ожидаемые данные
5) Interface Segregation Principle (Принцип разделения интерфейса).
- клиенты не должны быть вынуждены реализовывать методы, которые они не будут использовать.
- нужно делать интерфейсы под клиентов. Интерфейс должен содержать функционал для конкретной задачи клиента.
7) Dependency Inversion Principle (Принцип инверсии зависимостей).
- зависимости внутри системы строятся на основе абстракций
- Использовать все классы через интерфейсы
- Пример: можно создать arraylist или через экземпляр класса или через интерфейс:
  List<String> someList = new ArrayList<>();
  ArrayList<String> someList2 = new ArrayList<>();
- в 1 случае arraylist создан через интерфейс, во втором, через экземпляр класса
- Внутри класса прописать действия с абстракцией, В конструкторе передаём класс, который имплементирует интерфейс.
